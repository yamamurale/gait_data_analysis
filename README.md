# 下肢歩行周期・アシストタイミング・床反力ベクトル解析システム (Gait, Assist & Vector Analysis System)

## 1. 概要
本プロジェクトは、LabChartから取得した歩行時およびアシスト作動時の生データを統合的に解析し、歩行周期の特定、アシストタイミングの定量的評価、および床反力（GRF）ベクトルの高度な視覚化を行うためのPythonプログラム群です。
研究チーム内での円滑な共有（環境依存の排除）と共同研究の効率化を強く意識し、設定の共通化や、他のウェアラブルデバイスのログデータと容易に結合・同期できる高い拡張性を備えた設計にしています。
※本リポジトリでは、研究情報の機密保持および個人情報保護の観点から、一部のソースコードや実験データを省略・抽象化して掲載しています。

## 2. 目的・システムの特徴
研究効率の最大化（チーム共有の最適化）
  実験環境やPC環境が異なるチームメンバー間でもスムーズにコードを共有・実行できるよう、環境依存のパスやタスク定義を1つの設定ファイル（Config）に一元管理しています。
直感的かつ高度なデータ可視化（ベクトル図の描画）
  数値データだけでは捉えにくい歩行力学的な変化を、2次元のベクトル図（歩行周期ごとの床反力の向きと大きさ）として動的・静的に描画する機能を備えています。
高い拡張性（マルチモーダル統合を見据えた設計）
  将来的に他の生体センサ（筋電図など）やウェアラブルデバイスのログデータと時間軸を同期・結合して複合的な解析が行えるよう、関数群をモジュール化し、柔軟なデータ結合（Merge）が可能なデータ構造を採用しています。

## 3. ファイル構成と詳細

プロジェクトは以下の5つのモジュールから構成されています。

 ディレクトリ構造イメージ
├── Config.py              # 全体の共通設定・パス・タスク管理
├── Data_processing.py     # 解析用共通関数ライブラリ（信号処理・周期切り出し）
├── gaitcycle_force_labchart.py  # 【メイン1】歩行周期・床反力（GRF）解析
├── assist_analysis.py     # 【メイン2】外骨格ロボットのアシストタイミング解析
└── gaitcycle_vector.py    # 【メイン3】床反力ベクトルの描画・可視化

データ出力
 (タスク名)/             # 各タスクごとの出力フォルダ（自動生成）
    ├── 周期.csv
    ├── plot/              # GRF時系列プロットの格納先
    ├── assist/            # アシストタイミング解析結果・分布図の格納先
    └── vector/            # 床反力ベクトル図（片足・両足重ね合わせ）の格納先# gait_data_analysis

## 4.詳細
# Config.py（全体設定・環境管理）
役割: 解析全体におけるグローバルな設定を一元管理

# Data_processing.py（周期解析関数・信号処理ライブラリ）
役割: 信号処理および周期解析に必要な数理処理を行う共通関数ライブラリ

# gaitcycle_force_labchart.py（歩行周期解析メインプログラム）
役割: 歩行周期および床反力（GRF）の解析を行うメインプログラムの第一段階

# assist_analysis.py（アシストタイミング評価プログラム）
役割: 外骨格ロボット等のアシスト作動タイミングを定量評価・可視化するプログラム

# gaitcycle_vector.py（床反力ベクトル図描画プログラム）
役割: 算出した歩行周期およびアシストデータをもとに、床反力ベクトル図を描画・保存します。

## 5. 開発におけるこだわり・工夫した点
オブジェクト指向とモジュール化による保守性
頻繁に使用する信号処理や周期計算ロジック、データ読み込み処理を Data_processing.py に切り出したことで、メインコードの可読性を高め、バグの発生を大幅に抑制しました。

ユーザーインターフェースと柔軟性
gaitcycle_vector.py や gaitcycle_force_labchart.py では、コンソール上での対話型入力を採用し、解析したいタスク、グラフの保存有無、プロットモード（立脚期のみ/全周期、片足/両足重ね合わせ）を直感的に選択できるよう、実験ユーザーの利便性を最優先に設計しています。

データ構造の標準化とマルチモーダル展開
出力されるCSVファイルは、タイムスタンプや歩行周期（Gait Cycle %）を基準に標準化された構造にしています。これにより、今回は省略しているリハビリロボット側の制御ログや、その他のモーションキャプチャデータなどを後から容易に同期・結合（Merge）して、より複合的なデータ解析へと発展させることが可能です。

## 6. 使用言語・技術
言語: Python 3.x
主要ライブラリ:
NumPy, Pandas （データスクレイピング・数値計算・データ加工・時系列マージ）
SciPy （interpolateによる時間軸正規化、signalによるフィルタリング処理）
Matplotlib, Seaborn （高度なデータ可視化、ベクトルプロット、タイミング分布図の描画）
